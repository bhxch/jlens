# 贡献指南

感谢你对 JLens 的关注！我们欢迎各种形式的贡献，包括问题报告、功能建议以及代码提交。

## 开发环境

- **Java**: JDK 25 或更高版本。
- **构建工具**: Maven 3.9 或更高版本。
- **Node.js**: 用于测试和构建 npm 包。
- **Python**: 用于测试和构建 PyPI 包。

## 代码规范

- **格式化**: 使用 4 个空格缩进。
- **命名**: 遵循标准 Java 命名规范。
- **语言**: 代码、注释及文档请使用英文（Commit 信息除外）。
- **依赖**: 尽量减少依赖，仅使用已有的、稳定的库。

## 测试要求

- **单元测试**: 每个新功能或 Bug 修复都必须包含对应的单元测试。
- **覆盖率**: 核心逻辑的测试覆盖率需保持在 85% 以上。
- **集成测试**: 在提交更改前，确保所有集成测试均能通过。

## 提交规范

我们遵循结构化的 Commit 消息格式，以保持清晰的项目历史。

- **语言**: Commit 信息请使用**中文**。
- **格式**: `<类型>(<作用域>): <描述>`
- **单一职责**: 每个 Commit 应代表一个单一的逻辑变更。
- **常用类型**:
  - `feat`: 新功能
  - `fix`: Bug 修复
  - `docs`: 文档变更
  - `ref`: 代码重构
  - `test`: 增加或修正测试
  - `build`: 构建系统或分发相关的变更

## 分支模型

- 所有的 Pull Request 请提交至 `main` 分支。
- 在提交 PR 前，请确保你的分支已同步至 `main` 的最新状态。

## 文档结构

- **用户指南**: `docs/user-guide/`
- **开发者文档**: `docs/developer-docs/`
- **设计方案**: `docs/plans/`

当增加新工具时，请务必同步更新中英文文档。
